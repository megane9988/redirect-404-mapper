## Description
Redirect 404 Mapper automatically detects access to non-existent pages (404 errors) and redirects users to alternative URLs based on rules configured in the WordPress admin panel.

## Key Features
- Rule Management UI
- Dynamic Redirects
- Flexible URL Specification
- Secure Implementation

## Installation
1. Upload the `redirect-404-mapper` folder to `/wp-content/plugins/`
2. Activate the plugin from WordPress
3. Go to Settings > 404 Redirect Mapper to configure rules

## Changing Behavior  
To modify how redirects work, you can:
1. Add/edit rules in the admin panel
2. Customize the JavaScript redirect logic in the main plugin file
3. Use filters and hooks if needed
